## Fixes Still Needed

### 1. ~~Hardcoded `image/png` MIME type~~ — `routes.py` ✅

**Problem:** The inline image handler always saves the file as `.png` and uploads with `content_type="image/png"`,
regardless of what the model actually returned. Gemini can return `image/webp` or `image/jpeg`.

**Fix:** Derive extension and content type from `part.inline_data.mime_type`.

```python
import mimetypes

mime = part.inline_data.mime_type  # e.g. "image/webp", "image/png"
ext = mimetypes.guess_extension(mime) or ".bin"
filename = f"{uuid.uuid4().hex}{ext}"
filepath = IMAGE_CACHE_DIR / filename
filepath.write_bytes(image_bytes)
gcs_url = upload_to_gcs(f"images/{filename}", image_bytes, mime)
```

Also fix the `/api/images/{filename}` serving endpoint — it hardcodes `media_type="image/png"`:

```python
media_type, _ = mimetypes.guess_type(str(path))
return FileResponse(str(path), media_type=media_type or "application/octet-stream")
```

### 2. ~~No size cap on inline image bytes~~ — `routes.py` ✅

**Problem:** `part.inline_data.data` is written straight to disk and GCS with no size check.
A very large image could exhaust disk or GCS quota.

**Fix:** Add a size guard before writing.

```python
MAX_IMAGE_BYTES = 20 * 1024 * 1024  # 20 MB

image_bytes = part.inline_data.data
if len(image_bytes) > MAX_IMAGE_BYTES:
    logger.warning("[Image] Oversized inline image skipped (%d bytes)", len(image_bytes))
    continue
```

---

### 3. ~~Swallowed GCS exception~~ — `routes.py` ✅

**Problem:** The bare `except Exception:` for the GCS upload has no logging. GCS failures
are completely invisible in production — local fallback URLs appear with no indication of why.

**Fix:**

```python
except Exception as gcs_err:
    logger.warning("[Image] GCS upload failed, serving locally: %s", gcs_err)
    image_url = f"/api/images/{filename}"
```

## How Inline Image Data Works

When Gemini 3 Pro Image Preview generates an image, the ADK runner delivers it as a
`Part` with an `inline_data` field (not a `function_response`):

```
part.text          → prose narrative chunk
part.inline_data   → image generated by the model
  .mime_type       → "image/png" | "image/webp" | "image/jpeg"
  .data            → raw bytes (NOT base64 — already decoded by the SDK)
part.function_response → tool result (only generate_music now)
```

Handling order in the SSE loop:

```
event received
  ├─ part.text            → append to text buffer → flush to Firestore on threshold
  ├─ part.inline_data     → validate size → save bytes to disk → upload to GCS
  │                          → emit SSE {type: "image", url: gcs_url}
  │                          → log URL to Firestore via log_tool_call
  └─ part.function_response
       └─ generate_music  → emit SSE {type: "music", url: ...}
                             → log to Firestore via log_tool_call
```

Storage split:
- **GCS bucket** (`emotional-chronicler-assets`) — holds the raw image bytes, public URL
- **Firestore** (`interactions` array) — holds the URL string only, never binary data
- **Local disk** (`_image_cache/`) — fallback if GCS upload fails; also used for fast local serving
